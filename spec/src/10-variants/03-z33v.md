# Z33-V (Paging)

## Overview

The Z33-V extends the base Z33 with a **page table register** and hardware page-based address translation, providing fine-grained memory protection with per-page access control flags.

| Register | Description |
|---|---|
| `%pt` | Page table register — physical address of the page table in memory |

Unlike Z33-M and Z33-S, the Z33-V performs address translation in **both supervisor and user modes**.

## Page Table

The page table is stored in memory starting at the address in `%pt`. Each entry corresponds to one page and has the following format:

```
Bit:  ...  12   11   10   9    8    7    6..0
       ...  P    R    X    S    A    D    framenum
```

| Bit(s) | Name | Description |
|---|---|---|
| 6–0 | framenum | Physical frame number (7 bits, values 0–99) |
| 7 | D (Dirty) | 1 if the page has been modified |
| 8 | A (Accessed) | 1 if the page has been accessed |
| 9 | S (Supervisor) | 1 if access is restricted to supervisor mode only |
| 10 | X (Execute) | 1 if execution is allowed on this page |
| 11 | R (Read-only) | 1 if the page is read-only, 0 if read/write |
| 12 | P (Present) | 1 if the page is present in memory, 0 if invalid |


## Page Size

The page size is **100 cells**. With 10,000 total memory cells, there are 100 possible pages.

## Address Translation

For every memory access (in both supervisor and user mode) to logical address *λ*:

1. Compute the page number: *p* = ⌊*λ* / 100⌋
2. Compute the offset within the page: *o* = *λ* mod 100
3. Read the page table entry from memory at address `%pt` + *p*
4. Check the flags:
   - **P** must be 1 (page is present)
   - **R**: if the access is a write, R must be 0
   - **X**: if the access is an instruction fetch, X must be 1
   - **S**: if S is 1, the processor must be in supervisor mode
5. If all checks pass:
   - Compute physical address: *φ* = framenum × 100 + *o*
   - Set the **A** (Accessed) flag to 1
   - If the access is a write, set the **D** (Dirty) flag to 1
6. If any check fails: raise an *invalid memory access* exception and set the **FA** (Fault Address) register to *λ*

## Additional Register

| Register | Description |
|---|---|
| `%fa` | Fault Address — set to the logical address that caused a page fault |

The `%fa` register allows the exception handler to determine which address caused the fault, enabling implementation of demand paging and other virtual memory techniques.

## Properties

- Provides full virtual memory support with per-page protection.
- Translation occurs in both privilege modes (unlike Z33-M and Z33-S).
- The A and D flags enable the OS to implement page replacement algorithms (e.g., LRU approximation).
- The page table itself resides in physical memory and is not subject to translation (it is accessed by physical address via `%pt`).
