# Z33-S (Segmented Memory)

## Overview

The Z33-S extends the base Z33 with **10 segment descriptor pairs**, providing segmented memory with per-segment protection and relocation.

Each segment *s* (0 ≤ *s* < 10) has:

| Register | Description |
|---|---|
| `%base`*s* | Base physical address for segment *s* |
| `%lim`*s* | Limit (maximum offset within segment, exclusive) |

## Address Translation

- **Supervisor mode:** The segment registers are **not used**. Memory accesses use physical addresses directly.

- **User mode:** For every memory access to logical address *λ*:

  1. Compute segment number: *s* = ⌊*λ* / 1000⌋
  2. Compute offset within segment: *o* = *λ* mod 1000
  3. Check: is *o* in the range [0, `%lim`*s*)? (i.e., 0 ≤ *o* < `%lim`*s*)
  4. If **yes**: translate to physical address *φ* = `%base`*s* + *o*
  5. If **no**: raise an *invalid memory access* exception

## Properties

- Each of the 10 segments can be independently sized and placed in physical memory.
- Logical addresses 0–999 map to segment 0, 1000–1999 to segment 1, etc.
- Segments can be used to separate code, data, stack, and shared regions.
- The segment registers are writable only in supervisor mode.
- The fixed segment size of 1000 logical addresses per segment is an architectural constant.
