# Z33-M (Base/Limit MMU)

## Overview

The Z33-M extends the base Z33 with two additional registers for simple memory protection and relocation:

| Register | Description |
|---|---|
| `%base` | Base address added to all user-mode memory accesses |
| `%lim` | Limit — maximum allowed logical address (exclusive) |

## Address Translation

- **Supervisor mode:** The `%base` and `%lim` registers are **not used**. Memory accesses use physical addresses directly.

- **User mode:** For every memory access to logical address *λ*:

  1. Check: is *λ* in the range [0, `%lim`)? (i.e., 0 ≤ *λ* < `%lim`)
  2. If **yes**: translate to physical address *φ* = *λ* + `%base`
  3. If **no**: raise an *invalid memory access* exception

## Properties

- Provides **relocation**: user programs can be loaded at any physical address without modification.
- Provides **protection**: user programs cannot access memory outside their allocated region.
- Does **not** support sharing memory between processes (each process has a single contiguous region).
- The `%base` and `%lim` registers are writable only in supervisor mode.
